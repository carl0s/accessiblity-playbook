# Contributing

**This is a collaborative project — PRs are more than welcome!** 🙌

The goal is a living, multi-source accessibility knowledge base. If you know a great resource — a book, handbook, standard, guideline, research paper, or practical pattern — please help integrate it.

## Ways to contribute

- **Add a new source.** Distill it into a chapter under `chapters/`, tagged so it fits the cross-book framework in `SKILL.md`.
- **Improve the references.** Extend `glossary.md`, `patterns.md`, or `cheatsheet.md`.
- **Fix and clarify.** Corrections, clearer plain-language phrasing, and better cross-references are all valuable.

## Adding a new source

1. **Check the license first.** Only include material whose license permits redistribution. If you're unsure, open an issue before submitting a PR.
2. **Credit the author(s)** and **cite the work's license** in the "Credits & source licenses" section of the [README](README.md).
3. **Distill, don't dump.** Summarize into frameworks, mental models, and actionable guidance rather than copying large passages verbatim — this both respects the source and keeps the skill useful on demand.
4. **Tag your additions** so they slot into the existing book-tagged structure in `SKILL.md`.

## Style

- Plain language. Prefer short sentences and concrete guidance.
- Keep WCAG criteria, contrast ratios, and target sizes accurate and sourced.
- Cross-reference related concepts across books where it helps.

## Pull requests

- One focused change per PR where possible.
- Describe what you added and which source it comes from.
- Open an issue first if you want to discuss scope or a license question.

Thanks for helping make accessibility knowledge easier to reach. 💛
